### 1. 基础信息与变换

### ℹ️ 基础信息

- **Add**：添加标签 / 分类
- **Prim Path**：Prim 路径
    - 当前：`/World/GR1_T1/waist_roll/visuals` → 这是机器人腰部旋转关节视觉模型的唯一路径。
- **Instanceable**：可实例化
    - 状态：未勾选 → 表示该 Prim 是独立实例，不共享实例数据。

### 🔄 Transform（变换）

控制该 Prim 在场景中的位置、旋转和缩放：

- **Translate**（平移）：`X=0, Y=0, Z=0` → 位于局部坐标原点。
- **Orient**（旋转）：`X=0, Y=0, Z=0` → 无旋转（默认四元数）。
- **Scale**（缩放）：`X=1, Y=1, Z=1` → 1:1:1 原始比例。

---

### 2. 材质（Materials）

- **Prim**：当前关联的 Prim 路径。
- **Material Assignment**：材质绑定
    - 当前材质：`/World/GR1_T1/Looks/material_COCO0`
    - 这是赋予给该视觉模型的具体材质实例。
- **Strength**（强度）：`Weaker than Descendants`
    - 表示继承优先级：子物体材质优先级高于父物体。

---

### 3. 几何（Geometry）核心属性

这是控制网格（Mesh）外观与物理行为的关键区域：

### 🧱 Mesh（网格基础）

- **Normals**（法线）：控制光照渲染方向。
- **Orientation**（方向）：`rightHanded`（右手坐标系）→ 符合标准图形学规则。
- **Points**（点）：顶点坐标数据（只读，显示拓扑结构）。
- **Velocities / Accelarations**（速度 / 加速度）：用于物理模拟的动态数据。
- **Extent**（范围）：
    - `[(-0.120..., -0.139..., -0.031...), (0.105..., 0.140..., 0.351...)]`
    - 包围盒的最小 / 最大坐标，定义该几何体的空间占用大小。
- **Cast Shadows**（投射阴影）✅
    - 勾选后，该物体在光照下会投射阴影到地面 / 其他物体。
- **Shadow Terminator Fix**（阴影终结者修复）✅
    - 解决阴影连接处的失真问题（Shadow Terminator Artifact）。
- **Fast Refraction Shadow**（快速折射阴影）
    - 提升透明材质的阴影渲染效率。
- **Enable Rt SS Transmission**（启用光线追踪次表面透射）
    - 支持次表面散射（如皮肤、塑料材质）的光线追踪渲染。
- **Enable automatic LOD generation**（启用自动细节层次生成）✅
    - 自动生成不同精度的网格，远看时自动切换低模，提升性能。
- **Holdout Object**（遮挡物体）
    - 作为遮挡物，在渲染中屏蔽后方物体。
- **Invisible To Secondary Rays**（次级射线不可见）
    - 反射 / 折射光线时忽略该物体，优化性能。
- **Matte Object**（哑光物体）
    - 合成数据中该物体视为背景，不参与前景分割。
- **Multimatte ID**（多蒙版 ID）：`1`
    - 用于合成渲染中的多通道蒙版标识。

### ✂️ Face（面）

控制网格拓扑细分：

- **Indices / Counts**：面索引数据。
- **Linear Interpolation**：`cornersPlus1` → 边界插值规则。
- **Hole Indices**：孔洞索引。

### ⚙️ Refinement（细分）

- **Refinement Level**（细分级别）：`0` → 默认无细分。
- **Interpolate Boundary**：`edgeAndCorner` → 边界插值规则。
- **Subdivision Scheme**：`bilinear` → 双线性细分算法。
- **Triangle SubdivisionRule**：`catmullClark` → 卡特 - 克拉克细分算法（平滑细分）。

### 🎯 Corner / Crease（边角 / 折痕）

- **Indices / Sharpnesses**：控制边缘的锐利程度（不折、折角）。

---

### 4. 视觉属性（Visual）

- **Double Sided**（双面）：未勾选 → 仅渲染正面。
- **Single Sided**（单面）：未勾选。
- **Purpose**（用途）：`default` → 标准渲染用途。
- **Visibility**（可见性）：`inherited` → 继承父级可见性。
- **Display Color / Opacity**：显示颜色 / 透明度（用于编辑器预览，不影响最终渲染）。

# synthetic data recorder

![PixPin_2026-03-28_10-56-24.png](attachment:48f137d7-654b-476e-a358-0e1bac835877:PixPin_2026-03-28_10-56-24.png)

### 🎥 Synthetic Data Recorder（合成数据记录器）

这是 Isaac Sim 中用于**生成仿真数据集**的核心面板，可录制 RGB、深度、语义分割等多模态数据，用于训练深度学习模型。

---

### 1. Writer（写入器）

- 负责数据的底层写入逻辑，默认配置即可满足大部分场景需求，通常无需修改。

---

### 2. Render Products（渲染产物）

定义要录制的视觉数据类型与相机参数：

- **Camera Path**：相机 Prim 路径
    - 当前：`/OmniverseKit_Persp` → 使用默认透视相机。
- **X / Y**：输出图像分辨率
    - 当前：`512 × 512` → 生成 512×512 像素的图像。
- **Name**：产物名称（如 `rgb`, `depth`, `semantic`）
    - 用于区分不同类型的输出数据。
- **Add New Render Product Entry**：添加新的渲染产物
    - 可同时录制 RGB、深度、语义分割、边界框等多模态数据。

---

### 3. Parameters（参数，折叠状态）

- 包含**数据格式、压缩级别、传感器参数**等高级配置：
    - 如选择 PNG/JPEG 格式、设置深度图缩放、语义标签通道等。

---

### 4. Output（输出）

配置数据保存路径：

- **Working Directory**：工作目录
    - 当前：`/home/cyun/Project/isaac/IsaacLab` → 数据集根目录。
- **子目录**：`_out_sdrec` → 实际数据将保存在 `{Working Directory}/_out_sdrec/` 下。
- **S3 Bucket**（折叠）：云存储配置
    - 可将数据直接上传到 AWS S3 等云存储，适合大规模数据集生成。
- **Config**（折叠）：配置文件导出
    - 可导出当前录制配置为 JSON，方便复现或批量运行。

---

### 5. Control（控制）

管理录制流程与参数：

- **Number of frames**：要录制的总帧数
    - 当前：`0` → 表示无限录制，直到手动暂停。
- **RTSubframes**：实时子帧数
    - 当前：`0` → 每帧渲染的子步数，用于运动模糊或超采样。
- **Control Timeline**：控制时间线
    - 勾选后，录制进度与场景时间线同步，适合动画场景。
- **Verbose**： verbose 日志
    - 勾选后输出详细日志，便于调试。
- **Start / Pause**：开始 / 暂停录制
    - 点击 `Start` 开始生成数据，`Pause` 暂停录制。

# semantics schema editor

![PixPin_2026-03-28_10-57-51.png](attachment:5316156c-d871-4986-ad8c-1cc95c374e5d:PixPin_2026-03-28_10-57-51.png)

### 🧩 Semantics Schema Editor（语义标签编辑器）

---

### 1. Apply semantic data on selected objects（为选中对象应用语义数据）

手动为选中的 Prim 添加自定义语义标签：

- **New Semantic Type**：语义类型（如 `class`、`instance`、`material`）
    - 定义标签的类别，例如 `class` 表示物体类别，`instance` 表示实例 ID。
- **New Semantic Data**：语义值（如 `robot`、`table`、`wall`）
    - 具体的标签内容，与类型对应。
- **Add Entry On All Selected Prims**：为所有选中对象添加条目
    - 批量将「类型 + 值」应用到选中的多个 Prim。
- 针对单个 Prim：
    - 显示当前 Prim 路径（如 `/World/GR1_T1/waist_roll/visuals`）
    - 提示「This prim has no semantic data」表示当前无标签
    - **Add New Value**：为该 Prim 添加新的语义键值对

**2. Apply semantic data using prim names（通过 Prim 名称自动生成语义标签）**

批量根据 Prim 名称自动生成语义标签，适合快速标注大量资产：

|参数|作用|当前配置解析|
|---|---|---|
|**Label type**|语义标签类型（如 `class`）|当前为 `class`，表示类别标签|
|**New / Overwrite / Skip**|冲突处理策略|- `New`：新增标签- `Overwrite`：覆盖已有标签- `Skip`：跳过已有标签的对象|
|**Prim types filter**|过滤要处理的 Prim 类型|当前为 `Mesh, Material, Skeleton`，仅对网格、材质、骨骼对象应用标签|
|**Remove numerical ending**|移除名称末尾的数字|✅ 勾选，如 `robot_01` → `robot`|
|**Remove prefixes**|移除指定前缀|输入 `SM, MI, Mat`，如 `SM_robot` → `robot`|
|**Remove suffixes**|移除指定后缀|输入 `Mat, 6M`，如 `robot_Mat` → `robot`|
|**Apply cumulatively**|累积应用|✅ 勾选，保留已有标签，新增标签不覆盖旧数据|
|**Remove separators**|移除分隔符|✅ 勾选，如 `robot-arm` → `robotarm` 或 `robot_arm`（依规则处理）|
|**Preview**|预览效果|未勾选，勾选后可先查看处理结果再应用|
|**Apply to**|应用范围|- `Stage`：整个场景- `Selected`：仅选中对象|
|**Add / Remove / Remove All**|操作按钮|- `Add`：应用规则生成标签- `Remove`：移除选中对象的标签- `Remove All`：清空场景所有语义标签|

### 3. Output（输出，折叠状态）

- 用于查看语义标签的应用结果、日志或导出配置，通常包含：
    - 成功/失败统计
    - 生成的语义数据预览
    - 导出为 JSON/USD 等格式的选项
