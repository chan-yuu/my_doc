更新时间：2026-04-07

这篇不是教程，而是背诵版。目标只有一个：把和 AI / LLM / Agent 相关的高频基础概念压成一套可面试复述的框架。

## 一、主线

如果只保留一条主线，可以记成：

> AI → 机器学习 → 深度学习 → Transformer → LLM → RAG → Agent

这条线的含义是：AI 是总目标；机器学习是实现路径；深度学习是机器学习中的主流方法；Transformer 是当前大模型的核心架构；LLM 是基于 Transformer 训练出来的大语言模型；RAG 是给 LLM 外接知识；Agent 是让 LLM 具备规划、调用工具和执行能力。

所以 LLM 不是 Agent，RAG 也不是 Agent，它们只是 Agent 系统中的能力部件。

## 二、第一层：AI、机器学习、深度学习

### 什么是 AI

AI 不是某一种具体算法，而是让机器表现出智能行为这一类技术的总称。

### 什么是机器学习

机器学习不是把规则手写进去，而是让模型从数据中学习规律。一句话：传统编程是规则 + 数据 → 结果，机器学习是数据 + 结果 → 规则。

### 什么是深度学习

深度学习是用多层神经网络做表示学习的一类方法。它的核心不是"层数多"，而是"自动学习特征"。

### 监督学习、无监督学习、强化学习的区别

| 类型 | 核心特征 | 例子 |
| --- | --- | --- |
| 监督学习 | 有标签 | 分类、回归 |
| 无监督学习 | 无标签 | 聚类、降维 |
| 强化学习 | 通过奖励学习策略 | 游戏、控制、对齐训练 |

### 过拟合是什么

过拟合不是模型太强，而是模型把训练集记得太死，导致泛化能力下降。常见解决方式：增加数据、数据增强、正则化、Dropout、提前停止。

### 欠拟合是什么

欠拟合不是泛化差这么简单，而是模型连训练集都没有学好。它通常意味着模型能力不足，或者训练还不够。

### 过拟合和欠拟合的区别

一句话：欠拟合是没学会，过拟合是学得太死。

### 偏差和方差是什么

Bias 是模型假设太强，学不出真实规律。Variance 是模型对训练数据太敏感，换一批数据结果就波动很大。简单记成：高偏差更接近欠拟合，高方差更接近过拟合。

### 训练集、验证集、测试集的区别

| 数据集 | 作用 |
| --- | --- |
| 训练集 | 训练参数 |
| 验证集 | 调参、选模型 |
| 测试集 | 最终评估 |

测试集不是拿来反复调参的，否则测试集本身也会被污染。

### 分类和回归的区别

分类是预测离散标签，回归是预测连续数值。例如：邮件是否垃圾是分类，房价预测是回归。

## 三、第二层：神经网络与 Transformer

### 神经网络的本质是什么

神经网络本质上是一个可学习的函数逼近器。输入经过线性变换和非线性激活，逐层映射到输出空间。

### 什么是参数

参数就是模型需要学习的权重。参数量越大，不一定越强，但通常表示模型容量更大。

### 什么是训练

训练就是通过损失函数和梯度下降不断更新参数，让模型输出更接近目标答案。

### 什么是损失函数

损失函数衡量模型输出和真实答案之间的差距。训练的目标就是最小化损失。

### 什么是梯度下降

梯度下降不是"猜答案"，而是沿着损失下降最快的方向调整参数。

### Batch、Epoch、Step 是什么

| 概念 | 含义 |
| --- | --- |
| Batch | 一次送进模型的一小批样本 |
| Step | 参数更新一次 |
| Epoch | 整个训练集完整训练一轮 |

### 学习率是什么

学习率控制每次参数更新的步长。太大容易震荡，太小收敛太慢。

### 优化器是什么

优化器决定参数如何更新。最常见的是 SGD 和 Adam。工程上更常见的是 Adam，因为它通常更稳，也更容易收敛。

### 激活函数是什么

激活函数的作用是引入非线性。如果没有非线性，多层网络本质上仍然只是线性变换。常见激活函数：ReLU、GELU、Sigmoid、Tanh。

### 归一化是什么

归一化的作用不是"让数值好看"，而是让训练更稳定。深度学习里常见的是 Batch Normalization 和 Layer Normalization。

### Transformer 为什么重要

Transformer 不是第一个神经网络架构，但它解决了长距离依赖和并行训练的问题。当前 LLM 基本都建立在 Transformer 或其变体之上。

### Transformer 的核心是什么

如果只记三个词：Self-Attention、Multi-Head Attention、Feed Forward。其中最关键的是 Self-Attention。

### Self-Attention 是什么

Self-Attention 的作用是让序列中每个 token 都能看到其他 token，并根据相关性分配不同权重。一句话：它解决的是"当前 token 应该重点关注上下文中的哪些部分"。

### Q、K、V 是什么

可以这样记：Query 是我在找什么，Key 是我是什么特征，Value 是我真正提供什么信息。Attention 本质上是用 Q 和 K 算相关性，再对 V 加权求和。

### 位置编码为什么需要

因为 Attention 本身不天然包含顺序信息。如果没有位置编码，模型不知道"我爱你"和"你爱我"的顺序差异。

### Encoder-only、Decoder-only、Encoder-Decoder 的区别

| 架构 | 特点 | 代表方向 |
| --- | --- | --- |
| Encoder-only | 更偏理解 | BERT 类 |
| Decoder-only | 更偏生成 | GPT 类 |
| Encoder-Decoder | 输入输出分离 | T5、翻译、摘要 |

当前主流通用 LLM 大多是 Decoder-only。

### Mask 是什么

Mask 的作用是控制 token 能看到哪些位置。自回归生成里最常见的是 Causal Mask，也就是当前位置不能看到未来 token。

## 四、第三层：LLM 基础

### 什么是 LLM

LLM 是在海量文本上训练出来的下一 token 预测模型。它表面上是在"生成文本"，本质上是在做条件概率建模。

### 为什么说 LLM 是下一 token 预测

因为训练目标通常就是：给定前文，预测下一个最可能出现的 token。这件事做大之后，会涌现出问答、总结、翻译、代码生成等能力。

### token 是什么

token 不是单词，也不一定是字。它是模型处理文本时的基本离散单位。

### 上下文窗口是什么

上下文窗口是模型一次能看到的 token 范围。窗口越大，能处理的上下文越长，但计算成本通常也越高。

### 预训练是什么

预训练是在大规模通用数据上先学语言规律和世界知识。

### 指令微调是什么

指令微调不是教模型新知识，而是让模型更会按照人类任务形式输出答案。

### RLHF 是什么

RLHF 是 Reinforcement Learning from Human Feedback。核心思路是让模型根据人类偏好进一步对齐。

### SFT 和 RLHF 的区别

| 项 | SFT | RLHF |
| --- | --- | --- |
| 数据 | 标注好的输入输出对 | 偏好数据 / 奖励信号 |
| 目标 | 学会按任务回答 | 学会更符合人类偏好 |
| 本质 | 模仿学习 | 偏好优化 |

### 幻觉是什么

幻觉不是模型"胡说八道"这么简单，而是模型生成了形式上合理、事实上错误的内容。原因通常有三类：参数知识不完整、上下文不足、生成目标只要求"像"不保证"真"。

### 怎么降低幻觉

常见方法：更清晰的提示词、提供外部知识、RAG、工具调用、结构化约束、输出后校验。

### 参数量是不是越大越好

不是。参数量越大通常表示模型容量更大，但最终效果还取决于数据质量、训练方法、对齐方式和任务类型。

### 什么是涌现能力

涌现能力指的是模型规模变大之后，某些能力不是线性增强，而是突然变得可用。例如：多步推理、复杂指令跟随、代码生成。

### In-context Learning 是什么

In-context Learning 不是更新参数，而是模型在当前上下文里根据示例临时学会任务模式。Few-shot 的底层逻辑，本质上就是在利用 In-context Learning。

## 五、第四层：Prompt、推理与采样

### Prompt 是什么

Prompt 不是一句提示词，而是给模型构造决策上下文。更准确地说，Prompt 包括：角色设定、任务描述、输入数据、输出格式、约束条件。

### Zero-shot、Few-shot 是什么

| 类型 | 含义 |
| --- | --- |
| Zero-shot | 不给例子，直接做任务 |
| Few-shot | 给少量示例，让模型模仿模式 |

### Chain of Thought 是什么

Chain of Thought 不是让模型更聪明，而是把中间推理过程显式化。它对复杂推理任务往往有效。

### Temperature 是什么

Temperature 控制采样随机性。越高越发散，越低越稳定。

### Top-k 和 Top-p 是什么

Top-k 是只在概率最高的 k 个 token 里采样。Top-p 是只在累计概率达到 p 的 token 集合里采样。所以它们本质上都在控制输出的随机范围。

### 温度越低越好吗

不是。低温适合事实型、结构化任务。高温适合创意生成。关键是任务匹配，而不是一味调低。

### Beam Search 是什么

Beam Search 不是随机采样，而是保留多个高概率候选路径，再从中选择更优序列。它更偏"找最优路径"，而采样更偏"保留多样性"。

### System Prompt 为什么重要

System Prompt 决定的是模型的高层行为边界。它不是普通输入，而是对角色、目标、限制条件的全局约束。

## 六、第五层：Embedding、向量库、RAG

### Embedding 是什么

Embedding 不是压缩文本，而是把文本映射成稠密向量，使语义相近的内容在向量空间里更接近。

### 向量检索是什么

向量检索不是关键词匹配，而是语义相似度匹配。

### RAG 是什么

RAG 是 Retrieval-Augmented Generation。它不是训练模型，而是在生成前先检索外部知识，再把检索结果喂给模型。

### RAG 解决什么问题

核心解决三件事：参数知识过旧、私有知识无法写进参数、需要可追溯依据。

### RAG 的标准流程

如果只记最小流程，可以记成：切分 → 向量化 → 建索引 → 检索 → 重排 → 生成。

### Chunk 为什么重要

Chunk 太大，检索不准。Chunk 太小，语义不完整。所以 RAG 的效果很大程度取决于切分策略。

### Rerank 是什么

Rerank 不是再检索一次，而是对召回结果做更精细的相关性排序。

### RAG 的常见问题

检索不准、召回不全、Chunk 切分不合理、上下文拼接过长、检索到了但模型没用上。

### RAG 和微调的区别

| 项 | RAG | 微调 |
| --- | --- | --- |
| 作用 | 外接知识 | 改模型行为/能力 |
| 更新成本 | 低 | 高 |
| 知识更新 | 快 | 慢 |
| 适合场景 | 知识频繁变化 | 风格、任务模式稳定 |

一句话：RAG 更适合补知识，微调更适合改行为。

### 关键词检索和向量检索的区别

| 项 | 关键词检索 | 向量检索 |
| --- | --- | --- |
| 核心 | 词面匹配 | 语义匹配 |
| 优势 | 精确、可控 | 能处理同义表达 |
| 问题 | 泛化能力弱 | 可能不够精确 |

### 混合检索是什么

混合检索就是把关键词检索和向量检索结合起来。这样做的原因很简单：一个偏精确，一个偏语义。

### 查询改写是什么

查询改写不是让用户手动改问题，而是系统自动把原始问题改写成更适合检索的表达。

### RAG 一定要向量数据库吗

不一定。RAG 的本质是"先取回外部知识再生成"，向量数据库只是最常见的一种实现。

## 七、第六层：Agent 基础

### 什么是 Agent

Agent 不是单次问答，而是面向目标执行的闭环系统。

### Agent 的主体构成是什么

如果只保留最核心的四个部分：Plan、Tool、Memory、Action。

### Agent 和 LLM 的区别

LLM 解决生成问题。Agent 解决执行问题。

### Agent 和 Workflow 的区别

Workflow 更强调预定义流程。Agent 更强调动态决策。

### Agent 和 RAG 的关系

RAG 不是 Agent，但可以成为 Agent 的一个能力模块。更准确地说，RAG 是知识获取机制，Agent 是任务执行机制。

### Tool 是什么

Tool 是 Agent 连接外部世界的接口，例如：搜索、数据库、API、文件系统、Shell、浏览器。

### Memory 是什么

Memory 不是神秘概念，本质上就是状态管理。常见可分为：工作记忆（当前任务上下文）、情节记忆（历史轨迹）、语义记忆（规则、文档、知识）。

### Skill 是什么

Skill 是高于 Tool 的能力封装。它通常由 Prompt + Tool + 约束 + 流程 构成。

### Agent 的最小执行链

Goal → Plan → Act → Observe → Update。这条链要会背，因为它可以解释大多数 Agent 系统。

### 多 Agent 一定更好吗

不是。多 Agent 只有在任务天然可拆分、角色边界清晰时才成立。否则只是复杂度上升。

### Agent 为什么需要 Tool

因为参数知识是静态的，而真实世界是动态的。Tool 的作用不是锦上添花，而是把 Agent 从"只会说"变成"能做事"。

### Agent 和 MCP 的关系

MCP 不是 Agent，本质上是工具和资源接入协议。它解决的是 Agent 如何标准化连接外部能力。

### Agent 最难的工程问题是什么

通常不是 Prompt 本身，而是：状态管理、工具设计、权限控制、失败恢复、轨迹观测。

## 八、第七层：部署、推理、评估

### 训练和推理的区别

| 项 | 训练 | 推理 |
| --- | --- | --- |
| 目标 | 学参数 | 用参数 |
| 成本 | 高 | 相对低 |
| 是否更新模型 | 是 | 否 |

### 为什么推理成本高

因为生成是逐 token 进行的，尤其在长上下文和大模型下，算力开销很明显。

### 什么是量化

量化是把高精度参数压缩到更低精度，以换取更低显存和更高吞吐。

### 什么是蒸馏

蒸馏不是压缩参数本身，而是让小模型学习大模型的行为。

### 什么是 KV Cache

KV Cache 的作用是缓存历史 token 的 key/value，避免每生成一个新 token 都重复计算前文。

### 什么是吞吐和延迟

延迟是单次请求要等多久。吞吐是单位时间能处理多少请求或多少 token。一句话：用户更关心延迟，系统更关心吞吐。

### 什么是评估

评估不是只看"答案像不像对"，而是系统化衡量模型或 Agent 的效果。

### LLM 评估看什么

常见指标：准确性、一致性、幻觉率、格式遵循率、延迟、成本。

### Agent 评估为什么更难

因为 Agent 不只是输出文本，还会走轨迹、调用工具、修改状态。所以 Agent 评估要同时看：最终结果、中间轨迹、工具调用正确率、任务完成率。

### 什么是微调

微调不是从零训练大模型，而是在已有模型基础上继续训练，让它更适合特定任务或领域。

### 全参数微调和参数高效微调的区别

| 项 | 全参数微调 | 参数高效微调 |
| --- | --- | --- |
| 更新范围 | 几乎全部参数 | 少量新增或局部参数 |
| 成本 | 高 | 低 |
| 常见方法 | Full Fine-tuning | LoRA、Adapter、Prefix |

### LoRA 是什么

LoRA 的核心不是直接改原始大矩阵，而是用低秩矩阵近似增量更新。它的价值在于降低微调成本。

### 什么是对齐

对齐不是增加知识量，而是让模型行为更符合人类期望、规范和安全边界。

### 什么是 MoE

MoE 是 Mixture of Experts。它不是让所有参数同时激活，而是按输入只激活部分专家模块。

### 长上下文为什么难

长上下文不是把窗口拉大这么简单。真正难点在于：计算成本上升、注意力分散、远距离信息利用率下降、中间信息容易被忽略。

## 九、高频面试问答

### LLM 为什么有效

因为大规模数据、足够大的参数规模、Transformer 架构和自监督训练共同带来了能力涌现。

### 微调和 RAG 怎么选

如果问题是"知识不够"，优先 RAG。如果问题是"行为不对、格式不稳、领域表达不准"，再考虑微调。

### 为什么会有幻觉

因为模型优化的是"最可能的下一个 token"，而不是真实性本身。

### Attention 为什么比 RNN 更适合大模型

因为它更适合并行训练，也更容易建模长距离依赖。

### Agent 为什么比普通 LLM 更强

不是因为它"更聪明"，而是因为它能规划、能调用工具、能读反馈、能持续执行。

### RAG 的瓶颈通常在哪

多数时候不在模型，而在检索质量、切分策略、重排和上下文构造。

### 为什么说 Prompt Engineering 正在走向 Context Engineering

因为现在真正影响系统效果的，不只是提示词本身，而是给模型什么上下文、以什么结构给、在什么时机给。

### LangChain、LangGraph、LangSmith 分别是什么

LangChain 是通用 LLM 应用开发框架。LangGraph 是 Agent Runtime / 有状态执行层。LangSmith 是观测、调试、评估平台。

### MCP 是什么

MCP 是 Model Context Protocol。它解决的是模型或 Agent 如何以统一协议接入工具和资源。

### A2A 是什么

A2A 是 Agent 之间的互操作协议。如果说 MCP 解决"Agent 接工具"，那么 A2A 更接近"Agent 接 Agent"。

### 过拟合和欠拟合怎么判断

如果训练集效果就差，通常是欠拟合。如果训练集很好、验证集很差，通常是过拟合。

### 为什么 LoRA 现在这么常见

因为全参数微调成本太高，而 LoRA 能用更低的显存和训练成本完成任务适配。

### 向量检索是不是一定比关键词检索好

不是。向量检索更懂语义，但关键词检索在精确匹配、专业术语和结构化字段上通常更稳。

### Agent 的价值到底是什么

不是把对话做长，而是让模型具备规划、调用外部能力、读反馈、持续执行的能力。

### MCP 和 Function Calling 的区别

Function Calling 更像单次工具调用能力。MCP 更像工具和资源接入的标准协议层。前者偏模型能力，后者偏系统集成。

## 十、最后一版速记

如果只剩 1 分钟复习，记下面这些：

1. LLM 的本质是下一 token 预测。
2. Transformer 的核心是 Self-Attention。
3. 过拟合是学得太死，欠拟合是还没学会。
4. RAG 是外接知识，不是训练模型。
5. 微调是改行为，RAG 是补知识。
6. Agent 的本质是闭环执行，不是单次生成。
7. Agent 的主体构成是 Plan / Tool / Memory / Action。
8. Tool 让模型连上外部世界，Memory 让系统具备连续性。
9. LangChain 是应用框架，LangGraph 是 Agent Runtime。
10. MCP 管工具接入，A2A 管 Agent 协作。
11. LoRA 是低成本微调方法，MoE 是稀疏激活架构。
12. 面试里最重要的不是背术语，而是把边界讲清楚。
